Ruta de aprendizaje

Orden sugerido para una lectura sin lagunas.

1. Conceptos

Mock vs stub vs fake, patching, spies, y el “dónde parchear”.

2. Arquitectura

Cómo diseñar código mockeable: inyección de dependencias y límites de IO.

3. Librerías

unittest.mock, pytest-mock, monkeypatch, freezegun, HTTP mocking, AWS mocking.

4. FastAPI

TestClient, dependency_overrides, mock de servicios externos y auth.

5. Laboratorio

Práctica paso a paso: mockear un servicio externo en FastAPI sin tocar red.

6. Bases de Datos

SQLite in-memory, SQLAlchemy sessions, repositorios, y mocks sin fragilidad.

7. Fuentes

Documentación oficial y librerías recomendadas para ampliar y verificar.
Regla de oro: si tu test no falla cuando rompes el código, tu mocking está mal. Evita los “tests verdes que no testean nada” validando que el mock falla cuando toca.
