M7 — Debugging & Profiling

Encontrar cuellos de botella en asincronía es un arte distinto. Aprende a detectar quién está ralentizando tu Event Loop.

1. El Modo Debug de asyncio

Cuando el Event Loop va lento, generalmente significa que ALGUIEN ejecutó código bloqueante síncrono por accidente y acaparó el hilo principial. Para detectarlo, habilitamos el modo Debug.

python — Detectar lentitud automática
import asyncio
import time
import logging

# Configuramos logging para ver los warnings
logging.basicConfig(level=logging.WARNING)

async def tarea_molesta():
    # MAL: Esto bloquea el hilo por 0.5s y será captado por asyncio
    time.sleep(0.5)

async def main():
    loop = asyncio.get_running_loop()
    
    # Activamos el modo debug manualmente
    loop.set_debug(True)
    # (O puedes arrancar python con: python -X dev script.py)
    
    # Configuramos el umbral (por defecto es 0.1s, lo bajamos a 0.05s)
    loop.slow_callback_duration = 0.05
    
    await tarea_molesta()

# Output esperado:
# WARNING:asyncio:Executing <Task finished name='Task-1'...> took 0.501 seconds!
if __name__ == "__main__":
    asyncio.run(main())
¿Qué busca asyncio? Vigila cuánto tarda cada callback en terminar y devolver el control al loop. Si tardan más de 100 milisegundos (por defecto), lanza un WARNING a la consola. Este es tu faro en la oscuridad.

2. Profiling Profesional: yappi y aiomonitor

Las herramientas tradicionales como cProfile asumen que el tiempo de CPU es igual al tiempo transcurrido, pero en asincronía pasamos la mayor parte del tiempo esperando.

Yappi (Yet Another Python Profiler)

El único profiler diseñado de cero para entender greenlets y coroutines (asyncio). Separa claramente el tiempo de espera en Socket del tiempo real de CPU.

yappi.set_clock_type("wall")
yappi.start()
# tu codigo
yappi.stop()

aiomonitor

Abre un puerto TCP en tu aplicación viva y de producción. Permite conectarte por telnet/netcat y ver en tiempo real cuántas y cuáles tasks están atascadas.

with aiomonitor.start_monitor(loop):
  loop.run_forever()

3. Advertencias comunes y cómo resolverlas

"sys:1: RuntimeWarning: coroutine 'f' was never awaited"

Ocurre cuando llamas a una coroutine pero olvidaste el await delante. La tarea NO se ejecutó en absoluto.

res = fetch_data() # ❌ Lanza warning
res = await fetch_data() # ✅ Correcto

"Task exception was never retrieved"

Ocurre cuando creas una tarea con asyncio.create_task(), esa tarea sufre un Error interno, y como nadie hizo `await` sobre la tarea, el framework asyncio nunca supo a quién devolverle la excepción, así que simplemente la tira a la consola en formato warning.

Solución: Guarda las tareas en una lista y aségurate de hacer await asyncio.gather(*tasks) al final, o añádeles un callback task.add_done_callback(log_error_if_failed).
