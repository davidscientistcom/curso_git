M5 — Casos Reales TDD

Implementando un Crawler HTTP Concurrente y una Caché en Memoria Asíncrona con TDD estricto.

1. Caso Práctico: Crawler HTTP (httpx)

Queremos descargar 100 páginas web de forma asíncrona pero sin abrumar al servidor (usando un semáforo). Como siempre, empezamos por el test mockeando las llamadas HTTP usando respx y httpx.

python — test_crawler.py
import pytest
import httpx
import respx
import asyncio

# --- CÓDIGO PRODUCCIÓN ---
async def download_urls(urls: list[str], max_concurrent: int = 5) -> list[int]:
    sem = asyncio.Semaphore(max_concurrent)
    
    async def fetch(client, url):
        async with sem:
            response = await client.get(url)
            return response.status_code
            
    # Es vital reutilizar el mismo AsyncClient para usar connection pooling
    async with httpx.AsyncClient() as client:
        tasks = [fetch(client, url) for url in urls]
        return await asyncio.gather(*tasks)

# --- CÓDIGO DE TEST (TDD) ---
@pytest.mark.asyncio
async def test_download_urls_concurrently():
    urls = ["https://api.test/1", "https://api.test/2", "https://api.test/3"]
    
    # respx mockea httpx a nivel interno, sin usar AsyncMock manual
    with respx.mock:
        # Interceptamos todas las llamadas a api.test y devolvemos 200 OK
        respx.get(url__startswith="https://api.test/").mock(return_value=httpx.Response(200))
        
        statuses = await download_urls(urls, max_concurrent=2)
        
        # Verificamos
        assert len(statuses) == 3
        assert statuses == [200, 200, 200]

2. Caso Práctico: Async Cache con TTL

En aplicaciones web de alto rendimiento, si varios usuarios piden el mismo recurso a la vez mientras no está cacheado (Cache Stampede), podrías tumbar la base de datos de origen. La caché debe garantizar que solo el primero vaya a la BD y los demás esperen.

python — test_cache.py
import pytest
import asyncio
from unittest.mock import AsyncMock

# --- IMPLEMENTACIÓN DE LA CACHÉ ---
class AsyncCache:
    def __init__(self):
        self.store = {}
        # Usamos un Lock para evitar el Cache Stampede
        self.locks = {} 

    async def get_or_set(self, key: str, fetch_coro):
        # Si otra task ya está pidiendo esto, agarramos su lock (y esperamos)
        if key not in self.locks:
            self.locks[key] = asyncio.Lock()
            
        async with self.locks[key]:
            if key in self.store:
                return self.store[key]
            
            # Si no está, lo pedimos (tarda tiempo)
            result = await fetch_coro()
            self.store[key] = result
            return result

# --- EL TEST (Resolviendo el Cache Stampede) ---
@pytest.mark.asyncio
async def test_cache_stampede_prevention():
    cache = AsyncCache()
    
    # Este es nuestro mock de la BD. Queremos que SOLO SE LLAME 1 VEZ.
    mock_db_call = AsyncMock(return_value="Datos pesados")
    
    # Simulamos 50 usuarios pidiendo casi al mismo tiempo
    async def user_request():
        return await cache.get_or_set("clave_1", mock_db_call)
        
    tasks = [user_request() for _ in range(50)]
    results = await asyncio.gather(*tasks)
    
    # Todos deben recibir el dato
    assert all(r == "Datos pesados" for r in results)
    
    # LA MAGIA: El Lock garantizó que la BD solo se tocara UNA vez
    # Si usáramos concurrencia insegura, lo habría llamado 50 veces
    mock_db_call.assert_called_once()
    print("Cache Stampede prevenido ✅")
