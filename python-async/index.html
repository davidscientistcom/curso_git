Python Síncrono & Asíncrono

Apuntes expertos con enfoque TDD-First. El modelo de ejecución CPython, concurrencia real, asyncio de cero a producción, y cómo testear todo correctamente.

🎯
Enfoque TDD-First: Cada bloque de código va precedido de sus tests. Doble propósito: verificar la implementación y enseñar cómo testear código asíncrono, uno de los puntos más complicados del ecosistema Python. Requiere Python 3.11+ y pytest-asyncio 0.23+.

📚 Mapa del Curso

⚙️

M1 — GIL & Modelo de Ejecución

CPython por dentro, el GIL, concurrencia vs paralelismo vs asincronía, los tres modelos y sus reglas de uso.

 🔗

M2 — Python Síncrono con TDD

Threading, ThreadPoolExecutor, multiprocessing, ProcessPoolExecutor. Tests que demuestran race conditions, locks y benchmarks.

 ⚡

M3 — asyncio de Cero a Expert

Coroutines, event loop, gather, tasks, semaphore, queue, async context managers, async generators. Tests antes de cada concepto.

 🧪

M4 — Testing Async con pytest

pytest-asyncio completo: configuración, fixtures async, AsyncMock, timeouts, concurrencia. El módulo más crítico del curso.

 🏗️

M5 — Casos Reales TDD

Cliente HTTP concurrente, pipeline de datos, caché async con TTL, rate limiter para LLM APIs (OpenAI/Anthropic).

 🌐

M6 — Ecosistema Async

aiohttp, httpx, aiofiles, asyncpg, FastAPI sync vs async, asyncio.to_thread(), Celery vs asyncio.

 🔍

M7 — Debugging & Profiling

Modo debug asyncio, coroutines lentas, aiomonitor, yappi, errores de producción, memory leaks de tasks.
