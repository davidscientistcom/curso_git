1. El Bucle de Eventos (Event Loop) y Coroutines

¿Qué es el Event Loop? Es el componente central de asyncio: un bucle que gestiona y coordina la ejecución de muchas tareas (coroutines) en un solo hilo. El Event Loop mantiene una lista de tareas pendientes y, cuando alguna está lista para avanzar (por ejemplo, porque terminó de esperar un await), le da el turno para que siga ejecutándose. Así, con un solo hilo, puede gestionar miles de operaciones a la vez sin bloquearse.

En asyncio, el Event Loop nunca se olvida de ninguna tarea: cuando una operación asíncrona termina, el loop reanuda exactamente la coroutine que estaba esperando ese resultado, justo donde lo dejó.

Veamos ejemplos prácticos para entender cómo funciona:

python — Ejemplo 1: await en serie (NO concurrente)
import asyncio

async def tarea(id):
    print(f"Tarea {id}: inicio")
    await asyncio.sleep(1)
    print(f"Tarea {id}: fin")

async def main():
    await tarea(1)
    await tarea(2)
    await tarea(3)

asyncio.run(main())
# Salida:
# Tarea 1: inicio
# Tarea 1: fin
# Tarea 2: inicio
# Tarea 2: fin
# Tarea 3: inicio
# Tarea 3: fin
# (Tarda ~3 segundos)

En este ejemplo, aunque usamos async y await, las tareas se ejecutan una tras otra. El Event Loop espera a que termine cada una antes de empezar la siguiente. Esto es equivalente a código síncrono.

python — Ejemplo 2: await con gather() (Concurrente real)
import asyncio

async def tarea(id):
    print(f"Tarea {id}: inicio")
    await asyncio.sleep(1)
    print(f"Tarea {id}: fin")

async def main():
    await asyncio.gather(tarea(1), tarea(2), tarea(3))

asyncio.run(main())
# Salida:
# Tarea 1: inicio
# Tarea 2: inicio
# Tarea 3: inicio
# Tarea 1: fin
# Tarea 2: fin
# Tarea 3: fin
# (Tarda ~1 segundo)

Ahora sí, las tres tareas empiezan a la vez y el Event Loop las va avanzando según cada una llega a un await. El tiempo total es el de la tarea más lenta.

python — Ejemplo 3: create_task() y control manual
import asyncio

async def tarea(id):
    print(f"Tarea {id}: inicio")
    await asyncio.sleep(1)
    print(f"Tarea {id}: fin")

async def main():
    t1 = asyncio.create_task(tarea(1))
    t2 = asyncio.create_task(tarea(2))
    await asyncio.sleep(0.5)
    print("Haciendo otras cosas mientras las tareas corren...")
    await t1
    await t2

asyncio.run(main())
# Salida:
# Tarea 1: inicio
# Tarea 2: inicio
# Haciendo otras cosas mientras las tareas corren...
# Tarea 1: fin
# Tarea 2: fin

Con create_task() lanzamos tareas en background y podemos hacer otras cosas mientras tanto. El Event Loop gestiona todo el avance y reanuda cada tarea cuando corresponde.

⚖️
Una función definida con async def es una Coroutine. Cuando la llamas, no se ejecuta, ¡devuelve un objeto coroutine! Para ejecutarla realmente, debe ir acompañada de un await o enviarse al Event Loop con asyncio.run().
python — Ejemplo visual del Event Loop
import asyncio

async def tarea(id):
    print(f"Tarea {id}: inicio")
    await asyncio.sleep(1)
    print(f"Tarea {id}: fin")

async def main():
    await asyncio.gather(tarea(1), tarea(2), tarea(3))

asyncio.run(main())
# Salida:
# Tarea 1: inicio
# Tarea 2: inicio
# Tarea 3: inicio
# Tarea 1: fin
# Tarea 2: fin
# Tarea 3: fin
⚠️
TRAMPA CLÁSICA: await en serie = código síncrono disfrazado

El código de arriba parece asíncrono pero tarda 2 segundos en total.
¿Por qué? Porque await fetch_data(1) pausa main() y espera a que termine completamente antes de ni siquiera empezar fetch_data(2).

Para concurrencia REAL necesitas gather() o create_task(). Lo veremos en el bloque siguiente.

2. Ejecución Concurrente: Tasks y Gather

Hacer await secuencialmente derrota el propósito de asincronía. Para hacer cosas a la vez, necesitamos encolar las coroutines en el Event Loop como Tasks (Tareas).

🛠️ asyncio.create_task()

Programa una coroutine para que se ejecute en el fondo ("fuego y olvido", o guardas la referencia para hacer await más tarde).

📦 asyncio.gather()

El método rey. Le pasas N coroutines, las lanza todas concurrentemente y te devuelve una lista con los N resultados en el mismo orden original.

asyncio.gather()asyncio.create_task()
Cuándo usarlo Quieres esperar TODOS los resultados a la vez Quieres lanzar algo en background y seguir haciendo otras cosas
Devuelve Lista de resultados en el mismo orden en que los pasaste Un objeto Task al que puedes hacer await más tarde
Analogía Pedir varios platos y esperar que lleguen todos juntos Pedir un plato y mientras tanto hacer otra cosa en la mesa
💡
Ejemplo de create_task():
async def main():
    # Lanzamos la tarea en background SIN esperar
    task = asyncio.create_task(fetch_data(1))
    
    # Seguimos haciendo otras cosas mientras fetch_data(1) corre
    print("Haciendo otras cosas...")
    await asyncio.sleep(0.5)
    
    # Ahora sí esperamos el resultado
    resultado = await task
    print(resultado)

TDD: Verificando la concurrencia

Escribimos un test que verifica que la ejecución concurrente de 5 descargas de 0.5s tarda menos de 1 segundo.

python — test_gather.py
import asyncio
import time
import pytest

# Nota: usar el marcador pytest.mark.asyncio es OBLIGATORIO para tests async
@pytest.mark.asyncio
async def test_gather_execution_time():
    async def fake_request(id: int):
        await asyncio.sleep(0.5)
        return id
        
    start = time.time()
    
    # Lanzamos 5 peticiones A LA VEZ
    # Nota: pasamos coroutines directas: fake_request(1)
    resultados = await asyncio.gather(
        fake_request(1),
        fake_request(2),
        fake_request(3),
        fake_request(4),
        fake_request(5)
    )
    
    duration = time.time() - start
    
    # Verificamos que el orden se mantiene
    assert resultados == [1, 2, 3, 4, 5]
    # Verificamos concurrencia: 5x0.5s = 2.5s secuencial PERO <0.6s asíncrono
    assert duration < 0.6
    print(f"Tardó {duration:.2f}s ✅")

3. Semáforos (Limitando Concurrencia)

gather es genial, pero si le pasas 10,000 coroutines que hacen consultas a una BD, ¡quemarás el servidor! Necesitamos poner un "portero de discoteca". Eso es el Semaphore.

python — Semáforos
import asyncio

async def fetch_with_semaphore(sem: asyncio.Semaphore, id: int):
    # Si ya hay 3 personas dentro, el 4º se pausa aquí hasta que uno salga
    async with sem:
        print(f"[{id}] Entró. Consultando API...")
        await asyncio.sleep(1)
        print(f"[{id}] Salió.")
        return f"OK-{id}"

async def main():
    # Límite máximo: 3 coroutines ejecutando a la vez
    sem = asyncio.Semaphore(3)
    
    # Generamos 10 tareas
    tasks = [fetch_with_semaphore(sem, i) for i in range(10)]
    
    # Inicia. Verás en consola cómo entran de 3 en 3.
    await asyncio.gather(*tasks)
🔢
¿Qué ocurre con 10 tareas y Semaphore(3)? Veámoslo paso a paso:
  • Turno 1 → Tareas 0, 1, 2 entran y corren juntas (~1 segundo)
  • Turno 2 → Tareas 3, 4, 5 entran y corren juntas (~1 segundo)
  • Turno 3 → Tareas 6, 7, 8 entran y corren juntas (~1 segundo)
  • Turno 4 → Tarea 9 sola (~1 segundo)
  • TOTAL: ~4 segundos

Sin Semaphore → gather() lanzaría las 10 a la vez → ~1 segundo, pero 10.000 tareas tumbarían el servidor remoto.
Con Semaphore(3) → controlado, predecible, respetuoso con el servidor.

Regla práctica: Si vas a hacer más de ~50 peticiones concurrentes a una API o base de datos externa, siempre usa un Semaphore.
