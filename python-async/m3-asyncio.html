M3 — asyncio de Cero a Expert

Coroutines, The Event Loop, Tasks y Synchronization Primitives. Todo escrito sobre 1 solo hilo.

1. El Bucle de Eventos (Event Loop) y Coroutines

A diferencia de los hilos donde el Sistema Operativo te corta en cualquier momento (preemptive multitasking), en asyncio las funciones deciden cuándo pausarse voluntariamente y devolver el control al bucle central (cooperative multitasking).

⚖️
Una función definida con async def es una Coroutine. Cuando la llamas, no se ejecuta, ¡devuelve un objeto coroutine! Para ejecutarla realmente, debe ir acompañada de un await o enviarse al Event Loop con asyncio.run().
python — Sintaxis Básica
import asyncio

async def fetch_data(id: int) -> str:
    print(f"[{id}] Empezando descarga...")
    await asyncio.sleep(1) # Simulamos I/O. ¡Aquí es donde cedemos el control!
    print(f"[{id}] Descarga completa.")
    return f"Data {id}"

async def main():
    # Si hacemos 'await' uno tras otro, se vuelve SÍNCRONO (tarda 2s en total)
    res_1 = await fetch_data(1)
    res_2 = await fetch_data(2)
    print(f"Resultados: {res_1}, {res_2}")

# asyncio.run es la puerta de entrada.
# Crea el event loop, ejecuta la coroutine main(), y cierra el loop.
if __name__ == "__main__":
    asyncio.run(main())

2. Ejecución Concurrente: Tasks y Gather

Hacer await secuencialmente derrota el propósito de asincronía. Para hacer cosas a la vez, necesitamos encolar las coroutines en el Event Loop como Tasks (Tareas).

🛠️ asyncio.create_task()

Programa una coroutine para que se ejecute en el fondo ("fuego y olvido", o guardas la referencia para hacer await más tarde).

📦 asyncio.gather()

El método rey. Le pasas N coroutines, las lanza todas concurrentemente y te devuelve una lista con los N resultados en el mismo orden original.

🚨 TDD: Verificando la concurrencia

Escribimos un test primero que nos asegura que la ejecución de 5 descargas de 0.5s tarda menos de 1 segundo.

python — test_gather.py
import asyncio
import time
import pytest

# Nota: usar el marcador pytest.mark.asyncio es OBLIGATORIO para tests async
@pytest.mark.asyncio
async def test_gather_execution_time():
    async def fake_request(id: int):
        await asyncio.sleep(0.5)
        return id
        
    start = time.time()
    
    # Lanzamos 5 peticiones A LA VEZ
    # Nota: pasamos coroutines directas: fake_request(1)
    resultados = await asyncio.gather(
        fake_request(1),
        fake_request(2),
        fake_request(3),
        fake_request(4),
        fake_request(5)
    )
    
    duration = time.time() - start
    
    # Verificamos que el orden se mantiene
    assert resultados == [1, 2, 3, 4, 5]
    # Verificamos concurrencia: 5x0.5s = 2.5s secuencial PERO <0.6s asíncrono
    assert duration < 0.6
    print(f"Tardó {duration:.2f}s ✅")

3. Semáforos (Limitando Concurrencia)

gather es genial, pero si le pasas 10,000 coroutines que hacen consultas a una BD, ¡quemarás el servidor! Necesitamos poner un "portero de discoteca". Eso es el Semaphore.

python — Semáforos
import asyncio

async def fetch_with_semaphore(sem: asyncio.Semaphore, id: int):
    # Si ya hay 3 personas dentro, el 4º se pausa aquí hasta que uno salga
    async with sem:
        print(f"[{id}] Entró. Consultando API...")
        await asyncio.sleep(1)
        print(f"[{id}] Salió.")
        return f"OK-{id}"

async def main():
    # Límite máximo: 3 coroutines ejecutando a la vez
    sem = asyncio.Semaphore(3)
    
    # Generamos 10 tareas
    tasks = [fetch_with_semaphore(sem, i) for i in range(10)]
    
    # Inicia. Verás en consola cómo entran de 3 en 3.
    await asyncio.gather(*tasks)
