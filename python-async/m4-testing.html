M4 — Testing Async con pytest

Cómo mockear correctamente funciones asíncronas, testear bases de datos simuladas y usar pytest-asyncio.

1. Configuración de pytest-asyncio

Por defecto, pytest no sabe qué hacer si le pasas un test definido como async def (te dará un coroutine warning). Necesitamos la librería pytest-asyncio.

🔧
Configuración Crítica: En el archivo pyproject.toml o pytest.ini, añade asyncio_mode = "auto". Esto ahorra tener que poner @pytest.mark.asyncio encima de cada maldito test.

2. Fixtures Asíncronas (async fixtures)

A menudo necesitamos establecer una conexión a base de datos de tests antes de que empiece un test, y cerrarla al acabar.

python — conftest.py
import pytest
import asyncio

# Usar async def en una fixture requiere pytest-asyncio
# En vez de 'return', usamos 'yield' para poder limpiar recursos al terminar.
@pytest.fixture
async def db_connection():
    # ---- SETUP Phase ----
    print("Conectando a DB...")
    await asyncio.sleep(0.1) 
    fake_db = {"users": [1, 2, 3]}
    
    # Entregamos la DB al test
    yield fake_db 
    
    # ---- TEARDOWN Phase (se ejecuta tras acabar el test) ----
    print("Cerrando conexión DB...")
    await asyncio.sleep(0.1)
    fake_db.clear()

# El test inyecta la fixture de forma normal
async def test_database_has_users(db_connection):
    assert len(db_connection["users"]) == 3

3. El arte de mockear Coroutines: AsyncMock

Este es el punto que más dolores de cabeza da. Si tienes una función síncrona, usas MagicMock. Si tienes una función asíncrona y la mockeas con MagicMock, el test fallará con un error diciendo "objeto Mock no se puede hacer await".

La solución en Python 3.8+ viene de fábrica en la librería estándar: unittest.mock.AsyncMock.

python — test_mocking.py
from unittest.mock import patch, AsyncMock
import pytest

# --- Código de producción que queremos testear ---
async def check_user_exists(api_client, user_id):
    # Imaginemos que esta llamada tarda 5 segundos y cuesta $0.05 a la API..
    # ¡NO QUEREMOS LLAMARLA DE VERDAD EN UN TEST UNITARIO!
    response = await api_client.get_user(user_id)
    return response.get("status") == "active"

# --- El test usando AsyncMock ---
@pytest.mark.asyncio
async def test_user_exists_mock():
    mock_client = AsyncMock()
    
    # Decimos: "Cuando se haga await sobre get_user(123), devuelve este dict"
    mock_client.get_user.return_value = {"status": "active", "name": "Pepe"}
    
    # El código de prod hará "await mock_client.get_user(123)", lo que 
    # retornará instantáneamente lo que dijimos arriba.
    resultado = await check_user_exists(mock_client, 123)
    
    assert resultado is True
    # Verificamos que nuestro código sí llamó al método subyacente
    mock_client.get_user.assert_called_once_with(123)
💡
¿Sabías que...? Si usas @patch('modulo.api_client.get_user'), a partir de Python 3.8 python detecta si la función original era async def y automáticamente la reemplaza con un AsyncMock en lugar de un MagicMock normal. ¡Magia!
