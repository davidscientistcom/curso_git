M9 — Esquema Arquitectónico

El diagrama definitivo de toma de decisiones. ¿Cuándo debemos usar Threads, Multiprocessing o Asyncio?

Árbol de Decisión de Concurrencia en Python

Sigue las flechas para saber qué herramienta primitiva es la mejor para el cuello de botella de tu programa.

flowchart TD Start([¿Tu Tarea es lenta?]) --> Q1{¿Es CPU-Bound\no I/O-Bound?} %% Rama CPU Q1 -- "CPU-Bound\n(Cálculos matemáticos,\nprocesamiento ML)" --> CPU1["¿Llamas a librerías\nen C (NumPy,\nPandas, Polars)?"] CPU1 -- Sí --> GilLib["El GIL se libera en C"] --> Th1[Usa threading o\nThreadPoolExecutor] CPU1 -- No --> Pu[El GIL bloquea tu código\nPython] --> Mpe[Usa multiprocessing\no ProcessPoolExecutor] %% Rama I/O Q1 -- "I/O-Bound\n(Red, API, Base de Datos, Disco)" --> IO1{"¿Soporta la librería\nasincronía nativa\n(aiohttp, asyncpg)?"} IO1 -- No --> LegacyLib["Librería Bloqueante (requests, psycopg2)"] LegacyLib --> fastapi_q{"¿Estás en FastAPI?"} fastapi_q -- Sí --> Def[Usa 'def' normal\nFastAPI usará ThreadPool] fastapi_q -- No --> to_thread[Usa asyncio.to_thread()\no ThreadPoolExecutor] IO1 -- Sí --> ModernLib["Librería Async Nativa"] ModernLib --> AsyncDef[Usa asyncio puro\n(async def / await)] %% Estilos visuales classDef default fill:#1e1e2e,stroke:#4b5563,stroke-width:2px,color:#fff; classDef question fill:#3f3cbb,stroke:#8b5cf6,stroke-width:2px,color:#fff; classDef warning fill:#b45309,stroke:#f59e0b,stroke-width:2px,color:#fff; classDef success fill:#047857,stroke:#10b981,stroke-width:2px,color:#fff; class Q1,IO1,fastapi_q,CPU1 question; class LegacyLib,Pu warning; class AsyncDef,Mpe,Th1,Def,to_thread success;

Resumen rápido de los ganadores:
  • Para cálculos pesados puros de CPU en Python: multiprocessing (Procesos)
  • Para código síncrono antiguo bloqueante (I/O): concurrent.futures.ThreadPoolExecutor (Hilos)
  • Para I/O moderno y escalabilidad masiva: asyncio (Event Loop Cooperativo con Coroutines)
